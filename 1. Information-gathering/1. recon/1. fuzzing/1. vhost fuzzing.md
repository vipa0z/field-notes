
Using gobuster
```
gobuster vhost -u "http://10.129.239.24" -w /usr/share/wordlists/seclists/Discovery/DNS/subdomains-top1million-110000.txt --append-domain

-t 50 #threads \
-k flag can ignore SSL/TLS certificate errors.
-o flag to save the output to a file for later analysis.
```

Using ffuf
```
ffuf -u http://203.0.113.50 \
  -w /opt/useful/seclists/Discovery/DNS/subdomains-top1million-5000.txt:FUZ \
  -H "Host: FUZZ.example.com" \
```



filter response by size with `-fs`
filter status codes with `-fc`
